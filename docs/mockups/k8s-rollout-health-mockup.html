Collection Notes
To avoid hammering the control plane, the first version should derive most signals from: one workload list, one pod list, and current container status. “Restarted in the last 5 / 15 / 30 minutes” is only precise if we use each container’s lastState.terminated.finishedAt or current waiting/terminated timestamps. That avoids requiring broad event scraping as a first step.
If container status timestamps are missing or overwritten, we can degrade gracefully to: “has restarted recently” or “restart count increased since last refresh” once we decide whether in-memory sampling is acceptable.
