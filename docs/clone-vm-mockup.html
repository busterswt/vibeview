Boot volume clone path is available Cinder can create a new bootable volume from the current root volume.
!
Network identity cannot be duplicated exactly The clone should get a fresh Neutron port, MAC, and fixed IP by default even if it stays on the same tenant network.
!
Load balancer membership is detected but excluded A new clone should not silently join the same pool until the operator confirms backend readiness.
×
Floating IP is not auto-reusable The original floating IP remains attached to the source port; cloning it automatically would create an unsafe conflict.
